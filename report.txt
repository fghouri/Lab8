Lab Topic: Clients on a Router
Lab Description: 
    - We are going to model a wi-fi servers function when interacting with clients (everything but data transfer):
        - adding new clients (securely)
        - storing their rssi, connectivity statistics, name, radio bands ...etc
        - lookup of clients (for "sending" data)
        - deleting/ modifying client info
        - maintaining state
            - holding only set amount of clients based on their statistics, uptime, online status 
            - handle many clients added/ removed

2. Pros and cons of using classes (OOP) for this specific idea/problem
    Using classes is a good idea for storing and looking up client information.
    Building the client as a reusable object allows for us to manage many instances of them using the same         logic as well as standardize functions related to manipulating and adding/deleting them from our data        structures.
    Since the information related to each client differs and determines how it interacts on a network.
    As well the object can interact with the hashing part of the program to keep client information secure         while.
      it is being accessed and/or manipulated.
    
    Pros of using classes:
    - Allows us to store informations about a client in a well-ordered structure
        instead of scattered variables.
    - Convenient set/get functions.
    - Protect valuable data (private).

    Cons of using classes:
    - More memory

3. Data Structure used for this project? (unordered map?)
    The data structure used for this project is going to be Hash Table because it provides
    us with a time complexity of O(1) when handled properly. This project is going to take
    in multiple client's data and store it in bins in a hash table and it is important that
    the fetching speed is fast.
